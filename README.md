# Cafe Management System

## Overview
This is a **simple Cafe Management System** built using **Python and Flask**.  
The application handles client logins, authentication, and database management.  
It allows **admins** to add, edit, and delete products and staff, while **users** can access their own dashboards.

The project was designed as a learning experience to understand **Flask backend framework, authentication, authorization, and database management**.

---

## Goal of the Project
The main goals of this project were:

- Learn how to work with **Flask routes** and templates  
- Implement **authentication and authorization** for admins and users  
- Handle **secure logins and sessions**  
- Work with **databases** to store and manage products, staff, and users  
- Learn **CRUD operations** (Create, Read, Update, Delete)  

---

## Features

1. **Admin Dashboard** – view and manage products and staff  
2. **User Dashboard** – personalized for each user  
3. **Authentication Pages** – login and registration  
4. **Add/Edit/Delete** – manage tables, products, and staff  
5. **Tables** – organize cafe products and orders  
6. **File Uploads** – upload images for products  
7. **Error Handling** – ensure smooth user experience  

---

## What I Learned and Implemented

- **Flask Routes** – URL routing for pages and APIs  
- **API Integration** – fetch or handle data via requests  
- **Authentication** – login system using `Flask-WTF` forms  
- **Flask Sessions** – track logged-in users  
- **Authorization** – restrict access to admins and users  
- **Password Hashing** – using `werkzeug.security` for secure passwords  
- **Error Handling** – prevent crashes and show user-friendly errors  
- **File Uploads** – manage product images  
- **CRUD Operations** – create, read, update, delete records  
- **Database Models** – using SQLite3 and SQLAlchemy  
- **Templates and Jinja2** – dynamic HTML with inheritance  
- **HTTP Methods** – POST and GET requests  
- **Flask-Login** – user session management
